## What could be improved
Based on this experience there are a few key aspects that could have been improved upon in my design.
Firstly, whether I use a PIC or ESP32 in the future I would be sure to add both traces and male header pins
to each pin to allow for easier access in case of mistakes. Next I would be sure to add an overwhelming
amount of test points that would allow me to add extra components later if something were to break.
I would also like to re-organize the pcb into clearer sections so I can assemble and test a given
section before continuing onto the rest essentially working blind and praying the fully assembled 
board works. I would also spend more time selecting parts that are convienent for me to use and
not just components that are readily available. I prioritized parts we had in stock in class,
but those parts don't feel like they were the easiest choice when it came time to use them. I would
like to look into a different style of fuse holder as the one used felt cheap and unreliable
when compared to every other part on the board. Another issue I ran into on my schematic was relying
too much on some of the provided footprints by the manufacturer. Many of these had extra pins my
device did not have which got confusing down the line.
